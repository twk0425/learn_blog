<p>正则表达式是一种从左到右匹配主体字符串的模式。
<p>正则表达式:Regular expression，我们常使用缩写的术语“regex”或“regexp”

## 1. 基本匹配
<pre>
"the" => The fat cat sat on <a href="#learn-regex"><strong>the</strong></a> mat.
</pre>
<p>在日常生活中 ctrl+f 便可以快速进入到基本匹配模式，基本满足95%以上需求。

## 2. 元字符
<p>在一些场景下，我们需要从文本里找出符合某种规律的内容，这时就可以通过正则表达式来实现。
<p>正则表达式的核心是用“元字符”制定匹配规则，通过这些规则就能精准找到想要的文本模式。

### 2.1 穷举模式
<p> .:句号匹配任意单个字符除了换行符。
<p> [ ]:字符种类。匹配方括号内的任意字符。
<p> [^ ]: 否定的字符种类。匹配除了方括号里的任意字符。
<pre>
"[cf].t" => The <a href="#learn-regex"><strong>fat</strong></a> <a href="#learn-regex"><strong>cat</strong></a> sat on the mat.
</pre>

### 2.2 重复模式

<p> *:匹配 >=0 个重复的在 * 号之前的字符。
<p> +:匹配 >=1 个重复的 + 号前的字符。
<p> ?:标记 ? 之前的字符为可选，即出现 0 或 1 次。
<p> {n,m}:匹配 num 个大括号之前的字符或字符集 (n <= num <= m)。{,m}、{n,}、{num}分别表述小于等于m、大于等于n、等于num的匹配模式。

<pre>
"1{3}" => 3...6647093844940812848<a href="#learn-regex"><strong>111</strong></a>74502852110555
</pre>


### 2.3 特殊字符
<p> |:或运算符，匹配符号前或后的字符。
<p> \:转义字符，用于匹配一些保留的字符 <code>[ ] ( ) { } . * + ? ^ $ \ &#124;</code>
<p> ^:从开始行开始匹配。
<p> $:从末端开始匹配。

<p> \w:匹配所有字母数字，等同于 `[a-zA-Z0-9_]`
<p> \W:匹配所有非字母数字，即符号，等同于： `[^\w]`
<p> \d:匹配数字： `[0-9]`
<p> \D:匹配非数字： `[^\d]`
<p> \s:匹配所有空格字符，等同于： `[\t\n\f\r\p{Z}]`
<p> \S:匹配所有非空格字符： `[^\s]`
<p> \f:匹配一个换页符
<p> \n:匹配一个换行符
<p> \r:匹配一个回车符
<p> \t:匹配一个制表符
<p> \v:匹配一个垂直制表符
<p> \p:匹配 CR/LF（等同于 `\r\n`），用来匹配 DOS 行终止符

### 2.4 字符集
<p> (xyz):字符集，匹配与 xyz 完全相等的字符串。
<p>特征标群是一组写在 `(...)` 中的子模式。`(...)` 中包含的内容将会被看成一个整体，配合*+？{}等进行组合。
<p>在Python 或 JavaScript等宿主语言在，(...),(?:...)也称为捕获组、非捕获组。

## 3. 进阶
### 3.1 零宽度断言（前后预查）
用于筛选所有匹配结果，筛选条件为其前后 不跟随着断言中定义的格式
<p> ?=:正先行断言-存在
<p> ?!:负先行断言-排除
<p> ?<=:正后发断言-存在
<p> ?<!:负后发断言-排除

### 3.2 标志
标志也叫模式修正符，因为它可以用来修改表达式的搜索结果。
这些标志可以任意的组合使用，它也是整个正则表达式的一部分。
<p> i:忽略大小写。
<p> g:全局搜索。
<p> m:多行修饰符：锚点元字符 `^` `$` 工作范围在每行的起始。

<pre>
"/The/gi" => <a href="#learn-regex"><strong>The</strong></a> fat cat sat on <a href="#learn-regex"><strong>the</strong></a> mat.
</pre>
